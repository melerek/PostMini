Introduction

PostMini is a lightweight, desktop-based API client built with Python and PyQt6, designed to make testing and debugging HTTP APIs simple and efficient. Whether you're a backend developer testing your own endpoints or a frontend engineer exploring third-party APIs, PyRequester provides an intuitive interface for crafting HTTP requests and analyzing responses.

Unlike browser-based tools or heavyweight commercial applications, PostMini runs entirely on your local machine, giving you full control over your data and API interactions. All collections and requests are stored locally in a SQLite database, ensuring your work is persistent and easily accessible.

PostMini is ideal for developers who need a fast, reliable tool for working with REST APIs, without the complexity of enterprise-grade solutions. It's perfect for individual developers, small teams, and anyone who values simplicity and performance.

Key Features

PostMini offers a comprehensive set of features designed to streamline your API workflow:

  • Full HTTP Method Support: Send any type of HTTP request including GET, POST, PUT, DELETE, PATCH, HEAD, and OPTIONS.
  • Collection Management: Organize your API requests into logical collections for better project management and reusability.
  • Flexible Request Configuration: Add query parameters, custom headers, authentication tokens, and request bodies with ease.
  • Response Analysis: View beautifully formatted JSON responses, inspect response headers, and monitor status codes, response times, and payload sizes.
  • Local Persistence: All collections and requests are automatically saved to a local SQLite database, so your work is never lost.
  • Postman Import: Seamlessly migrate your existing Postman collections using the built-in import feature.
  • Git-Based Collaboration: üåü NEW! Version control collections with Git for seamless team collaboration.
  • Environment Variables: Use variables across your requests for different environments (dev, staging, production).
  • Request History: Track and replay all your previous API calls with full request and response details.
  • Code Generation: Generate code snippets in multiple languages (Python, JavaScript, cURL, C#, etc.) from your requests.
  • OAuth 2.0 Support: Built-in OAuth 2.0 flow support for secure API authentication.
  • API Testing: Define assertions and run automated tests on your API responses.

The Main Interface

PostMini features a clean, three-pane interface inspired by popular API clients but optimized for desktop use. The layout is designed to maximize productivity while maintaining clarity and ease of use.

Left Pane: Collections

The Collections pane on the left side of the window displays all your saved collections and requests in a hierarchical tree view. Each collection can contain multiple requests, making it easy to organize related API calls together. You can expand or collapse collections to manage your workspace efficiently.

Click on any collection to select it, or click on a specific request to load it into the Request Editor. The number of requests in each collection is displayed in brackets next to the collection name.

Top-Right Pane: Request Editor

The Request Editor occupies the top portion of the right side. This is your primary workspace for building and configuring HTTP requests. At the top, you'll find the method selector and URL input field, along with Send, Save, and Code Generation buttons.

Below the URL bar, a tabbed interface provides organized access to different request components:

  • Params: Add query parameters that will be appended to your URL.
  • Headers: Define custom HTTP headers for your request.
  • Authorization: Configure authentication (Bearer Token, Basic Auth, OAuth 2.0).
  • Body: Specify the request payload for POST, PUT, and PATCH requests.
  • Tests: Define automated assertions to validate API responses.

Bottom-Right Pane: Response Viewer

The Response Viewer displays the results of your API calls. After sending a request, this pane will populate with comprehensive response information. At the top, you'll see the HTTP status code (color-coded for quick recognition), response time, and payload size.

The response content is organized into tabs:

  • Body: The response body, automatically formatted if it's JSON.
  • Headers: A table showing all HTTP headers returned by the server.

You can resize the panels by dragging the divider between the Request Editor and Response Viewer to suit your preferences.

Managing Collections

Collections are the foundation of organizing your API work in PostMini. They allow you to group related requests together, making it easy to switch between different projects or API endpoints.

Creating a New Collection

  1. Click the "Add Collection" button located at the top of the Collections pane on the left side of the window.
  2. Enter a descriptive name for your collection in the dialog box that appears. Choose a name that clearly identifies the project or API group (e.g., "GitHub API", "Payment Gateway", "User Service").
  3. Press Enter or click OK to create the collection. It will immediately appear in your Collections tree view.

Adding Requests to a Collection

  1. Select the collection where you want to add a new request by clicking on it in the tree view.
  2. Click the "Add Request" button in the Collections pane. The dialog will show which collection you're adding to.
  3. Enter a name for the request that describes its purpose (e.g., "Get User Profile", "Create Order", "Delete Comment").
  4. Click OK and the request will be created with default settings (GET method, example URL). The Request Editor will automatically open, ready for you to configure.

Deleting Collections or Requests

  1. Select the item to delete by clicking on it in the Collections tree. You can select either a collection or an individual request.
  2. Click the "Delete" button or press the Delete key on your keyboard.
  3. Confirm the deletion when prompted. Note that deleting a collection will also delete all requests within it.

üí° Pro Tip: Collections stay collapsed by default. When you add or modify requests, PostMini remembers which collections were expanded, maintaining your workspace layout.

Building a Request

Once you've selected or created a request, you can configure it using the various components of the Request Editor. Each aspect of your HTTP request can be customized to match your API's requirements.

URL & Method

The URL bar and method selector are located at the top of the Request Editor and form the foundation of your API call.

Selecting the HTTP Method

Click the dropdown menu on the left side of the URL bar to choose your HTTP method. The available options are:

  • GET: Retrieve data from the server (most common for read operations).
  • POST: Send data to create a new resource on the server.
  • PUT: Update an existing resource completely.
  • PATCH: Partially update an existing resource.
  • DELETE: Remove a resource from the server.
  • HEAD: Retrieve headers only, without the response body.
  • OPTIONS: Check which methods are supported by the endpoint.

Entering the URL

Type or paste your API endpoint URL into the text field. This should be the complete URL including the protocol (http:// or https://). For example: 

https://api.example.com/v1/users

You can use environment variables in your URLs by wrapping them in double curly braces: 

{{base_url}}/api/users/{{user_id}}

Press Enter while in the URL field to quickly send the request without clicking the Send button.

Parameters

Query parameters are key-value pairs appended to your URL after a question mark (?). They're commonly used to filter, sort, or paginate API responses.

Adding Parameters

  1. Click the "Params" tab in the Request Editor.
  2. Enter the parameter key in the "Key" column (e.g., "page", "limit", "search").
  3. Enter the parameter value in the "Value" column (e.g., "1", "10", "John").
  4. Add more rows as needed for additional parameters. PostMini automatically adds these to your URL when you send the request.

For example, if your base URL is https://api.example.com/users and you add parameters page=2 and limit=50, the final URL will be: 

https://api.example.com/users?page=2&limit=50

Headers

HTTP headers provide additional information about your request or specify how the server should process it. Common headers include Content-Type, Accept, and custom authentication headers.

Adding Headers

  1. Switch to the "Headers" tab in the Request Editor.
  2. Enter the header name in the "Key" column (e.g., "Content-Type", "Accept", "X-API-Key").
  3. Enter the header value in the "Value" column (e.g., "application/json", "text/html").

Some commonly used headers:

  • Content-Type: Specifies the format of the request body (e.g., application/json).
  • Accept: Tells the server what format you want in the response.
  • User-Agent: Identifies your client application.
  • Custom Headers: Many APIs use custom headers like X-API-Key or X-Request-ID.

Authorization

Many APIs require authentication to verify your identity. PyRequester supports multiple authentication methods through the Authorization tab.

Using Bearer Token Authentication

  1. Navigate to the "Authorization" tab.
  2. Select "Bearer Token" from the "Type" dropdown menu.
  3. Paste your token into the "Token" input field. This is typically a long string provided by the API you're accessing (e.g., a JWT token).
  4. PostMini will automatically add the Authorization header with the format Bearer YOUR_TOKEN when you send the request.

Other Authentication Types

PostMini also supports:

  • No Auth: For public endpoints that don't require authentication.
  • Basic Auth: Username and password encoded in Base64.
  • OAuth 2.0: Full OAuth 2.0 flow support with automatic token management.

üîí Security Note: Your authentication tokens are stored locally in the PostMini database. Never share your database file or commit it to version control systems.

Body

The request body contains data you're sending to the server. This is primarily used with POST, PUT, and PATCH requests when you need to create or update resources.

Adding a Request Body

  1. Open the "Body" tab in the Request Editor.
  2. Enter your data in the text area. For JSON APIs, format your data as a valid JSON object.
  3. Make sure to set the Content-Type header to application/json in the Headers tab if you're sending JSON data.

Example JSON body for creating a new user:

{
  "username": "johndoe",
  "email": "john@example.com",
  "firstName": "John",
  "lastName": "Doe",
  "role": "developer"
}

PyRequester supports any text-based format in the body, including JSON, XML, plain text, or form data.

Sending Requests

Once you've configured all aspects of your request, you're ready to send it to the API server.

How to Send a Request

  1. Click the "Send" button located next to the URL bar, or press Ctrl+Enter on your keyboard for faster workflow.
  2. Watch the button change to "‚è≥ Sending..." while your request is being processed. This visual feedback lets you know PostMini is actively communicating with the server.
  3. View the response in the Response Viewer pane below once the request completes. The button will change to "‚úì Send" to indicate success.

‚å®Ô∏è Keyboard Shortcut: Press Ctrl+Enter from anywhere in the Request Editor to quickly send your request without reaching for the mouse.

Saving Your Requests

After configuring a request, click the "Save" button (or press Ctrl+S) to persist your changes. PostMini will indicate unsaved changes with an asterisk (*) next to the request name, and will prompt you before switching to another request if you have unsaved changes.

Viewing the Response

After sending a request, the Response Viewer displays comprehensive information about the server's response.

Response Metadata

At the top of the Response Viewer, you'll see three key metrics:

  • Status Code: The HTTP status code returned by the server (e.g., 200, 404, 500). The status is color-coded for quick recognition:
    • Green (2xx): Success
    • Blue (3xx): Redirection
    • Orange (4xx): Client Error
    • Red (5xx): Server Error
  • Time: The total time taken for the request to complete, measured in milliseconds. This helps you identify slow endpoints and performance issues.
  • Size: The size of the response payload in bytes or kilobytes. Useful for monitoring data transfer and API efficiency.

Response Body

The Body tab displays the main content returned by the server. PostMini automatically detects and formats JSON responses for better readability, with proper indentation and syntax highlighting.

For non-JSON responses (HTML, XML, plain text), the content is displayed as-is in the viewer.

You can use the search functionality within the response body to quickly find specific values or keys. The "Copy Response" button allows you to copy the entire response to your clipboard for use in other applications.

Response Headers

Switch to the Headers tab to view all HTTP headers returned by the server. These headers provide metadata about the response, such as:

  • Content-Type: The format of the response body.
  • Content-Length: The size of the response in bytes.
  • Date: When the response was generated.
  • Server: Information about the server software.
  • Custom Headers: Any application-specific headers the API returns.

Headers are displayed in a clean table format with key-value pairs for easy scanning.

Request History

Every request you send is automatically saved to your request history. Click the "üìã History" button in the toolbar to view past requests, including their full configuration and responses. You can replay any historical request with a single click.

Importing from Postman

If you're migrating from Postman or want to share collections between tools, PostMini offers seamless import functionality that's fully compatible with Postman Collection Format v2.1.

Exporting from Postman

  1. Open Postman and navigate to the collection you want to export.
  2. Click the three dots (‚ãØ) next to the collection name and select "Export".
  3. Choose "Collection v2.1" as the export format. This is the recommended format and ensures maximum compatibility.
  4. Save the JSON file to a location on your computer that you can easily access.

Importing into PostMini

  1. Open PostMini and look for the "Import Collection" button in the Collections pane on the left side.
  2. Click "Import Collection" to open the file selection dialog.
  3. Navigate to and select the JSON file you exported from Postman.
  4. Click "Open" and PostMini will automatically:
    • Detect the Postman format
    • Convert it to PostMini's internal format
    • Create a new collection with all your requests
    • Preserve request methods, URLs, headers, bodies, and parameters
  5. Verify the import by expanding the new collection in the tree view and checking a few requests to ensure everything imported correctly.

üîÑ Two-Way Compatibility: PostMini can also export collections in Postman format, making it easy to share your work with team members who use different tools. Use the "Export Collection" button and select the "Postman v2.1" format option.

What Gets Imported?

The import process preserves all essential request components:

  • Collection name and structure
  • Request names and descriptions
  • HTTP methods (GET, POST, PUT, DELETE, etc.)
  • Complete URLs including variables
  • Query parameters
  • Request headers
  • Request bodies (JSON, form data, raw text)
  • Basic authentication settings

Known Limitations

While PostMini strives for full compatibility, some advanced Postman features may not import perfectly:

  • Pre-request scripts and test scripts (Postman's JavaScript code)
  • Complex folder hierarchies (converted to flat structure)
  • Postman-specific variables and environments (must be recreated)

For these advanced features, you may need to manually reconfigure them using PostMini's native environment variables and testing capabilities.

üåü Git-Based Collaboration

PostMini includes powerful Git-based collaboration features that enable seamless team workflows. Version control your API collections alongside your code, review changes via Pull Requests, and keep your team's API knowledge synchronized.

How It Works

PostMini syncs your collections to a .postmini/ folder as JSON files. You commit these files to Git, and your teammates can pull and import them. It's a three-layer system:

  • PostMini Database: Your local SQLite database with all collections
  • .postmini/ Files: JSON export of your collections (committed to Git)
  • Git Repository: Version control and sharing mechanism

‚ö†Ô∏è Important: PostMini syncs between the database and files. YOU still run Git commands (git commit, git push, etc.) to share changes with your team. PostMini is a file sync tool, not a Git client.

Getting Started

  1. Click the "üîÑ Git Sync" button in the toolbar at the top of PostMini.
  2. Select your project directory (the root of your Git repository).
  3. Enable auto-sync to automatically export collections when you save.
  4. Click "üöÄ Enable Git Sync" to initialize the .postmini/ folder.
  5. Commit to Git: Run git add .postmini/, git commit -m "Add API collections", and git push.

Team Workflow

Here's how a typical team collaboration scenario works:

  1. Engineer A creates collections in PostMini and clicks Save (auto-exports to .postmini/).
  2. Engineer A commits to Git: git add .postmini/ && git commit -m "Add User API" && git push
  3. Engineer B pulls from Git: git pull
  4. Engineer B imports in PostMini: Opens PostMini, clicks "üîÑ Git Sync" ‚Üí "Status" ‚Üí "üì• Import from Files"
  5. Collections are now in sync! Both engineers have the same API collections.

Key Benefits

  • PR Reviews: Review API changes like code changes with Git diffs
  • Version Control: Track who changed what and when
  • Branch-Based Development: Use Git branches for API development
  • Secure Secrets: Secrets are automatically separated and gitignored
  • Works Offline: No cloud services required
  • Any Git Hosting: GitHub, GitLab, Bitbucket, or self-hosted

Secrets Management

PostMini automatically detects sensitive variables (like apiKey, authToken, password) and separates them:

  • Public config (in Git): Contains placeholders like {{'{{'}}SECRET_PROD_APIKEY{{'}}'}}
  • Local secrets (gitignored): Contains actual secret values in .postmini-secrets/

Each team member maintains their own secrets locally. No risk of accidentally committing API keys!
