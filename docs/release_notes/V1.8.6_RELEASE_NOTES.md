### Temporary Tabs (Preview Mode) ğŸ†•

**Single-click vs Double-click behavior:**
- **Single-click** on a request in Collections â†’ Opens in **temporary tab** (italic text)
- **Double-click** on a request â†’ Opens in **permanent tab** (normal text)
- **Only one temporary tab** can exist at a time
- Temporary tab converts to permanent when you:
  - **Edit the request** (start typing)
  - **Send the request**
  - **Save the request**
  - **Double-click** to explicitly make it permanent

**Benefits:**
- Browse requests quickly without cluttering tabs
- Preview requests before committing to open them
- More efficient tab management for large collections
- Similar to VS Code's file preview behavior

### Add Collection Variables from Variables Panel ğŸ”§

**New Capability:**
- Add variables directly to collections from the Variables panel
- Dropdown menu on **"+ Add"** button with two options:

## ğŸ› Bug Fixes

### Fixed Recent Button Hover Cutoff
- **Problem:** Hover effect was cut off at the bottom of the button
- **Cause:** Fixed 38x38px size with emoji caused rendering issues
- **Solution:** Converted to flexible text-based button with proper sizing

### Improved Signal Blocking During Load
- **Fixed:** Request loading now properly blocks signals to prevent false "modified" state
- **Added:** Signal blocking for test_tab and scripts_tab
- **Result:** Loading saved requests no longer marks them as modified
- **Impact:** Prevents accidental unsaved changes prompts

---

### History Panel Mutual Exclusion
- **Fixed:** History and Recent Requests panels now properly hide each other
- **Result:** Only one right-side overlay visible at a time
- **Impact:** Cleaner UI, no overlapping panels
